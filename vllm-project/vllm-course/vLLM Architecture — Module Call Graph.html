A

离线推理路径 — LLM.generate()

从 Python API 调用到 token 生成的完整链路
▸
LLM.generate(prompts, sampling_params)
vllm/entrypoints/llm.py
calls →
用户入口。验证参数，添加请求，启动引擎循环。
LLMEngine.add_request(request_id, prompt, params)
vllm/v1/engine/llm_engine.py
calls →
InputProcessor 构建 EngineCoreRequest，注册到 OutputProcessor。
InprocClient.add_request(request) → EngineCore.add_request()
vllm/v1/engine/core_client.py → core.py
calls →
请求进入调度器的等待队列。前缀缓存哈希计算。
LLMEngine.step() → InprocClient.get_output()
vllm/v1/engine/llm_engine.py → core_client.py
calls →
循环执行直到所有请求完成。每次调用 EngineCore.step()。
EngineCore.step() → scheduler.schedule()
vllm/v1/engine/core.py → v1/core/sched/scheduler.py
calls →
调度器决定哪些请求获得 GPU 时间，分配 KV cache 块。
executor.execute_model(scheduler_output)
vllm/v1/executor/abstract.py → Worker.execute_model()
calls →
通过 collective_rpc 调度 Worker，Worker 调用 GPUModelRunner。
GPUModelRunner.execute_model() → _model_forward() → model(input_ids, positions, ...)
vllm/v1/worker/gpu_model_runner.py → model_executor/models/*.py
calls →
准备输入 → 构建注意力元数据 → 执行模型前向传播 → 计算 logits。
GPUModelRunner._sample(logits) → Sampler(logits, sampling_metadata)
vllm/v1/worker/gpu_model_runner.py → v1/sample/sampler.py
calls →
温度缩放 → Top-K/Top-P → 惩罚 → Gumbel 采样 → 新 token。
scheduler.update_from_output(scheduler_output, model_output)
vllm/v1/core/sched/scheduler.py
returns →
更新请求状态，追加生成的 token，检查停止条件，构建 EngineCoreOutputs。
OutputProcessor.process_outputs() → RequestOutput
vllm/v1/engine/output_processor.py
反 tokenize，构建最终的 RequestOutput 返回给用户。

C

引擎核心循环 — EngineCore.step()

vLLM 的心跳 — 每个 token 都来自这个循环的一次迭代
▸
scheduler.has_requests() → 检查是否有待处理请求
vllm/v1/core/sched/scheduler.py
if True →
如果无请求，直接返回空结果。
scheduler_output = scheduler.schedule()
vllm/v1/core/sched/scheduler.py
→ SchedulerOutput
遍历 running 队列 → 分配 token budget → KVCacheManager.allocate_slots() → 构建批次。
future = executor.execute_model(scheduler_output, non_block=True)
vllm/v1/executor/abstract.py
async →
异步提交给 GPU Worker 执行。非阻塞返回 Future。
grammar_output = scheduler.get_grammar_bitmask(scheduler_output)
vllm/v1/core/sched/scheduler.py
parallel →
与 GPU 执行并行计算结构化输出约束的 bitmask。
model_output = future.result()
vllm/v1/engine/core.py
→ ModelRunnerOutput
等待 GPU 完成。如果 model_output 为 None，调用 sample_tokens()。
engine_core_outputs = scheduler.update_from_output(scheduler_output, model_output)
vllm/v1/core/sched/scheduler.py
追加 sampled tokens → 检查停止条件 → 处理推测解码拒绝 → 释放完成请求的块 → 返回 EngineCoreOutputs。
D

模型前向传播 — GPU 上发生的事

从 SchedulerOutput 到 sampled token 的 GPU 执行路径
▸
Worker.execute_model(scheduler_output)
vllm/v1/worker/gpu_worker.py
calls →
每个 GPU 上的 Worker 接收调度输出，委托给 ModelRunner。
GPUModelRunner._update_states(scheduler_output)
vllm/v1/worker/gpu_model_runner.py
calls →
更新 InputBatch：添加新请求，更新 block tables，设置 slot mappings。
GPUModelRunner._prepare_inputs() → input_ids, positions, attn_metadata
vllm/v1/worker/gpu_model_runner.py
calls →
从 InputBatch 提取当前步的 input_ids 和位置编码，构建注意力元数据。
GPUModelRunner._model_forward(input_ids, positions, attn_metadata, ...)
vllm/v1/worker/gpu_model_runner.py
calls →
设置 forward_context → 调用 self.model(input_ids, positions, ...) → 可能使用 CUDA Graph。
model(input_ids, positions, ...) — e.g. LlamaForCausalLM.forward()
vllm/model_executor/models/llama.py (或其他模型)
calls →
Embedding → N × (Attention + MLP) → 最终 hidden states。注意力使用 PagedAttention / FlashAttention。
model.compute_logits(hidden_states) → logits
vllm/model_executor/models/*.py
calls →
最后一层 hidden states → lm_head 线性层 → vocabulary 大小的 logits 向量。
GPUModelRunner._sample(logits, spec_decode_metadata)
vllm/v1/worker/gpu_model_runner.py
calls →
普通采样 → Sampler()；推测解码 → RejectionSampler()。输出 SamplerOutput。
propose_draft_token_ids(sampled_token_ids) [if spec decode]
vllm/v1/worker/gpu_model_runner.py
如果启用推测解码，在采样后立即让 Drafter 生成下一步的候选 token。
E

KV Cache 管理路径

从调度器请求块到 GPU 块表的内存管理链
▸
Scheduler.schedule() → 遍历 running 请求
vllm/v1/core/sched/scheduler.py
calls →
对每个请求计算需要多少新 token，然后请求 KV cache 块。
KVCacheManager.allocate_slots(request, num_new_tokens)
vllm/v1/core/kv_cache_manager.py
calls →
计算需要多少新块，查找前缀缓存命中，委托给 Coordinator。
KVCacheCoordinator.allocate_new_blocks(num_blocks)
vllm/v1/core/kv_cache_coordinator.py
calls →
协调多个 KV cache 组，委托给 SingleTypeKVCacheManager。
BlockPool.get_new_blocks(num_blocks) → list[KVCacheBlock]
vllm/v1/core/block_pool.py
returns →
从 free_block_queue 分配。如果不够，触发 LRU 逐出。返回物理块 ID。
SchedulerOutput 携带 block_ids → GPUModelRunner._update_states()
vllm/v1/core/sched/output.py → gpu_model_runner.py
data flow →
新请求的 block_ids 存入 CachedRequestState，更新 GPU 端 BlockTables。
BlockTables.update(block_ids) → GPU tensor
vllm/v1/worker/gpu/block_table.py
→ Attention
slot = block_number × page_size + offset。注意力内核使用 block tables 读写 paged KV cache。
